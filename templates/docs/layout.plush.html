Introduction

Welcome to the Loan Service API documentation. This API allows you to integrate loan management functionality into your applications.

Base URL

https://api.loanservice.com/v1

Request Format

All requests should be made with JSON payloads and include the appropriate headers:

Content-Type: application/json
Authorization: Bearer YOUR_API_KEY

Authentication

All API requests require authentication using Bearer tokens.

Getting an API Key

Contact your system administrator to get your API credentials.

Using the API Key

Include your API key in all requests using the Authorization header:

Authorization: Bearer YOUR_API_KEY

‚ö†Ô∏è Important Security Notes:

  • Keep your API key secure and never share it
  • Don't expose the API key in client-side code
  • Use different API keys for development and production

Step by Step Guide: How to Process a Loan

Prerequisites

Before starting, make sure you have:

  • Your API key for authentication
  • Borrower's personal information
  • Required documents ready to upload

Phase 1: Initial Setup

Step 1: Create a Borrower Account

First, register the borrower in the system

POST /api/v1/borrowers 
curl -X POST "https://api.loanservice.com/api/v1/borrowers" \
-H "Authorization: Bearer YOUR_API_KEY" \
-H "Content-Type: application/json" \
-d '{
    "name": "John Doe",
    "email": "john@example.com",
    "phone": "+1234567890",
    "address": "123 Main St",
    "credit_score": 750
}'

üìù Save the borrower_id from the response - you'll need it for creating the loan

Phase 2: Loan Creation

Step 2: Create a Loan Application

Create a loan using the borrower_id from Step 1

POST /api/v1/loans 
curl -X POST "https://api.loanservice.com/api/v1/loans" \
-H "Authorization: Bearer YOUR_API_KEY" \
-H "Content-Type: application/json" \
-d '{
    "borrower_id": "BORROWER_ID_FROM_STEP_1",
    "principal_amount": 50000,
    "rate": 12.5,
    "roi": 8.5,
    "term_months": 24
}'

üìù Save the loan_id from the response - you'll need it for all following steps

Phase 3: Document Collection

Step 3: Upload Required Documents

Upload all necessary documents for the loan

POST /api/v1/loans/{loan_id}/documents 
curl -X POST "https://api.loanservice.com/api/v1/loans/LOAN_ID/documents" \
-H "Authorization: Bearer YOUR_API_KEY" \
-H "Content-Type: multipart/form-data" \
-F "document=@/path/to/document.pdf" \
-F "type=identity_proof" \
-F "description=Identity verification document"

‚ö†Ô∏è Repeat this step for each required document (ID proof, income proof, etc.)

Phase 4: Loan Approval

Step 4: Get Loan Approved

Field validator must visit borrower and submit approval

PUT /api/v1/loans/{loan_id}/approve 
curl -X PUT "https://api.loanservice.com/api/v1/loans/LOAN_ID/approve" \
-H "Authorization: Bearer YOUR_API_KEY" \
-H "Content-Type: multipart/form-data" \
-F "field_validator_id=EMP456" \
-F "approval_date=2024-03-21" \
-F "verification_photo=@/path/to/photo.jpg" \
-F "remarks=Borrower verification completed successfully"

‚úÖ After approval, the loan becomes visible to investors

Phase 5: Investment Collection

Step 5: Collect Investments

Multiple investors can invest in the loan

POST /api/v1/loans/{loan_id}/investments 
curl -X POST "https://api.loanservice.com/api/v1/loans/LOAN_ID/investments" \
-H "Authorization: Bearer YOUR_API_KEY" \
-H "Content-Type: application/json" \
-d '{
    "investor_id": "INV789",
    "amount": 25000,
    "investment_date": "2024-03-21"
}'

‚ö†Ô∏è Repeat this step until total investment equals loan amount

Step 5.1: Check Investment Status

Verify if the loan is fully funded

GET /api/v1/loans/{loan_id}/investments/status 
curl -X GET "https://api.loanservice.com/api/v1/loans/LOAN_ID/investments/status" \
-H "Authorization: Bearer YOUR_API_KEY" \
-H "Content-Type: application/json"
